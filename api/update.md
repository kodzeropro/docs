# Update

Обновление существующей записи по идентификатору.

## Запрос

```http
PATCH /v1/:project/:collection/:id
Content-Type: application/json
```

| Параметр | Описание |
|----------|----------|
| `:id` | Уникальный идентификатор записи (`_id`) |

## Тело запроса

JSON-объект с полями, которые нужно обновить. Можно передавать только изменяемые поля:

```json
{
  "price": 899
}
```

## Пример запроса

```http
PATCH https://api.kodzero.pro/v1/:project/:collection/:id
Content-Type: application/json

{
  "price": 899
}
```

## Ответ

```json
{
  "ok": true,
  "result": {
    "_id": "abc123",
    "title": "iPhone 15",
    "price": 899,
    "createdAt": "2025-01-15T10:30:00Z",
    "updatedAt": "2025-01-18T14:20:00Z"
  }
}
```

::: info
Поле `updatedAt` обновляется автоматически при каждом изменении записи.
:::

## Частичное обновление

Метод `PATCH` выполняет частичное обновление — передавайте только те поля, которые нужно изменить. Остальные поля сохранят свои значения.

```http
# Обновить только цену
PATCH /v1/:project/:collection/:id

{
  "price": 799
}
```

## Ошибки

### Запись не найдена

```json
{
  "ok": false,
  "error": "<string>"
}
```

**HTTP статус:** `404`

### Ошибка валидации

```json
{
  "ok": false,
  "error": "<string>"
}
```

**HTTP статус:** `400`
