# Логи запросов

Kodzero ведёт журнал всех API-запросов к вашему проекту. Это помогает отслеживать активность, находить ошибки и анализировать использование.


![Logs example](/logs.png)

::: info <IconBeta/>Хранение логов (Retention)
По умолчанию логи хранятся в течение 7 дней. В будущем планируется увеличить срок хранения для пользователей платных тарифов.
:::

## Просмотр логов

Логи доступны во вкладке **«Логи»** в панели управления проектом. Здесь отображаются все запросы ко всем коллекциям с возможностью фильтрации.

## Структура записи

Каждая запись лога содержит:

| Поле | Описание |
|------|----------|
| `_id` | Уникальный идентификатор записи лога |
| `startedAt` | Время начала запроса |
| `collection` | Идентификатор коллекции |
| `method` | HTTP-метод (GET, POST, PATCH, DELETE) |
| `originalUrl` | Полный путь запроса |
| `status` | HTTP-статус ответа |
| `spentMs` | Время выполнения в миллисекундах |

## Пример записи

| _id | startedAt | collection | method | originalUrl | status | spentMs |
|-----|-----------|------------|--------|-------------|--------|---------|
| 6a7b8c9d... | 2025-01-18 12:30:45 | products | GET | /v1/proj123/products?page=1 | 200 | 45 |
| 7b8c9d0e... | 2025-01-18 12:31:02 | products | POST | /v1/proj123/products | 201 | 82 |
| 8c9d0e1f... | 2025-01-18 12:31:15 | orders | GET | /v1/proj123/orders/abc123 | 404 | 12 |

## Фильтрация логов

В настоящий момент доступна фильтрация по коллекции — выберите нужную коллекцию из выпадающего списка, чтобы увидеть только её запросы.

## Применение

### Отладка ошибок

Статусы <strong>4xx</strong> подсвечены жёлтым цветом, а <strong>5xx</strong> — красным для быстрого обнаружения проблем.

Найдите запросы с проблемным статусом, чтобы понять причину проблем.

### Анализ производительности

Отслеживайте поле `spentMs`, чтобы выявить медленные запросы. Запросы дольше <strong>500мс</strong> подсвечены жёлтым, а дольше <strong>1000мс</strong> — красным.

<i>:pushpin: Пороговые значения могут быть изменены в дальнейшем.</i>
